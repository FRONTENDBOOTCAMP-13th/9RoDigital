윌리의 서재 서비스 이용하기

가이드

윌리의 서재 UI는 HTML, Tailwind CSS를 활용한 컴포넌트 방식으로 제작되었습니다. 각 요소는 재사용성과 확장성을 고려하여 분리되었습니다.

모듈식 레이아웃

헤더, 푸터, 버튼, 타이틀 등 레이아웃 요소를 독립적으로 관리하고 조합하여 사용할 수 있습니다.

Tailwind 기반 디자인

일관된 디자인 시스템으로 빠른 스타일링이 가능하며, 커스터마이징이 용이합니다.

반응형 디자인

모바일부터 데스크탑까지 모든 화면 크기에 최적화된 레이아웃을 제공합니다.

간편한 사용법

HTML 파일에 컴포넌트 요소를 포함하는 방식으로 간편하게 페이지를 구성할 수 있습니다.

컴포넌트 조립 방법

1. 사전 준비 단계

1.1 디자인 시안 분석

전체 레이아웃과 구조를 면밀히 검토합니다. 각 섹션별 필요한 컴포넌트를 식별합니다. 컴포넌트 간 관계를 파악합니다.

1.2 컴포넌트 확인

components 폴더 내 모든 컴포넌트 목록을 확인합니다. 각 컴포넌트의 역할과 기능을 검토합니다. 필요한 컴포넌트가 없다면 새로 생성하거나 기존 컴포넌트 수정을 고려합니다.

2. 컴포넌트 조립 프로세스

2.1 기본 구조 설정

먼저 메인 레이아웃을 구성합니다. 전체 페이지 또는 애플리케이션의 기본 구조를 정의합니다.

2.2 컴포넌트 배치

디자인 시안의 레이아웃 순서대로 컴포넌트를 배치합니다.
